Address lookup and USA tax matching

Google Places and USPS help identify the customer's address, ZIP, ZIP+4, city and state. The local ZIP/tax coverage database is a separate required layer for matching that ZIP/state to imported USA tax rate records used during checkout.

Address autocomplete alone does not load tax rates and does not enable tax calculation by itself. Tax calculation needs imported tax data connected to the product, service, category, subcategory or tax group. If Google, USPS or local coverage is unavailable, address forms keep working normally without suggestions.

Google Places address autocomplete

Google Places shows the fast dropdown under Street address after the customer starts typing. It fills address fields when the customer selects a suggested address.

USPS address and ZIP verification

USPS verifies the selected or typed address and improves ZIP, ZIP+4, city and state matching. ZIP-only city/state updates use USPS when this service is enabled and configured.

USPS setup links: USPS Developer Portal  |  USPS setup guide  |  Addresses API 3.0 docs

Google Places sends typed address text to Google before form submission. Use it only if your privacy policy allows this. USPS lookup is for checkout/address ZIP help tied to mailing or shipping workflows. Do not use either provider as a tax-rate source or as legal tax advice.

USA tax source states

Select states and load official-source parser output into the lower staging table. The source list contains 50 states plus DC; territories are not included until explicitly supported. Use the starter button to quickly stage all supported normalized examples.

Daily source updates

The global setting Auto-update USA taxes daily only enables the scheduler hook. Each live tax row in the main Taxes table must also have Auto-update this tax from source enabled. POSMall then refreshes that row from its official source when the source changes.

Tax data responsibility

This USA tax helper is only a quick setup aid. POSMall does not guarantee that these rates, tax groups, source links or parser results are complete, current, legally correct or available at all times. Tax compliance responsibility remains with the store owner. Review official sources and get qualified tax or legal advice before collecting or filing taxes.

Some Washington 5-digit ZIP codes contain multiple ZIP+4 rate candidates. POSMall stores a dominant 5-digit ZIP fallback for checkout convenience, but exact Washington local tax can require a full address or ZIP+4 lookup against the official DOR address, ZIP+4 and rate files.

For USA checkout, delivery/shipping address is the primary tax location for shipped goods. Billing address should only be used as a fallback when no delivery address or place-of-use address is available.
