Laravel scheduler

To enable scheduled runs, configure the Laravel scheduler on your server:

* * * * * cd /path/to/project && php artisan schedule:run >> /dev/null 2>&1

This backend page stores which tests should run and when. The scheduler executes the saved plan.

PostgreSQL catalog load benchmark

This local tool creates synthetic POSMall-owned catalog rows, rebuilds the PostgreSQL index, runs category/filter/search benchmarks, and stores the result below. It does not run during normal installation or normal seeding.

Before visual or image-enabled benchmark runs, rebuild the optimized image cache. Original photos stay untouched; generated WebP/JPEG derivatives are stored under the POSMall cache tree.

Image-enabled benchmark seeding creates product image-set records in the database and reuses optimized image derivatives instead of copying one binary photo per product.

Disabled outside local development

Catalog load benchmarks are available only in local/dev/testing environments.

Large runs use the console

Use the buttons for normal local checks. For 200,000 and 300,000 products, run the benchmark from the console so the backend page does not wait on a long web request.

php artisan posmall:images:optimize-catalog --profile=all
php artisan posmall:pagespeed:optimize-assets
php artisan posmall:load-benchmark 200000 --iterations=10 --force --with-images
php artisan posmall:load-benchmark 300000 --iterations=10 --force --with-images
